A brief introduction

Scholarly.ML is designed to allow researchers to discover colleagues with similar research interests through the semantic analysis of arxiv abstracts. Scholarly doesn't tackle the pure literature discovery problem like Semantic Scholar - it instead chooses to focus on giving researchers the chance to find interesting colleagues to collaborate with as well as discover some insights about the research that currently exists within ML.

How about you search for an interesting ML researcher like David Sontag or for an interesting Generative Adversarial Network paper?

It's probably worth noting the following:
  • More features are planned - but given university deadlines, updates will be slow! Current ideas include:
    • An 'Author Graph' do discover your immediate research network
    • Clustering and graphical analysis to determine the seminal paper - without citations
    • Integrations with Citations/PDF analysis
  • All the processing was done in an entirely unsupervised manner with absolutely no labels
  • Current drawbacks that will need to be addressed include:
    • Using a static dataset with data up to July 2017. Making it dynamic will require infrastructure that will cost :(
    • The dataset is exclusively scraped from arxiv.org and due to computing restrictions, is restricted to purely ML/AI papers
    • The server is heavily RAM limited due to the large dataset sizes and AWS Free Tier restrictions
    • Semantic analysis is only based on the abstracts provided on arxiv.org - currently no other data is being pulled
    • The underlying model used tends to bias towards very general concepts as opposed to 'novel' research topics. This makes it difficult to flag truly novel papers. Doing this will require some sort of rethink.
